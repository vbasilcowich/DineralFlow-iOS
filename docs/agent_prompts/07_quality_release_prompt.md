# Prompt 07 - Quality and Release

## Objetivo

Preparar el repo para validacion continua y para el salto posterior a pruebas reales en iPhone o Mac.

## Limites

- no abrir una rama de release aun
- no automatizar App Store

## Entregables

- tests base
- checklist de validacion
- estrategia para Expo Go, build cloud o Mac

## Criterio de exito

- el equipo sabe que puede validar en Windows y que queda pendiente para macOS

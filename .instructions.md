# DineralFlow iOS - Lean Agent Instructions

This repo is an Expo/React Native iOS client for DineralFlow with a low-cost Cloudflare backend. Optimize for small context, small patches, and fast verification.

## Instruction Budget

- Read the minimum set first: target file, one adjacent dependency, one relevant test.
- Use `rg` to locate symbols before opening files.
- Do not preload `docs/`, `lib/`, `components/`, or repo trees.
- Open extra files only when blocked by missing context.
- Prefer existing repo docs over web search unless the task depends on current external facts, pricing, or platform setup.
- Load domain-specific file hints from `docs/ai_lookup.md` only when needed.

## Default Workflow

1. Inspect the file being changed.
2. Inspect the nearest hook, API client, type, or shared component that actually affects it.
3. Make the smallest working change.
4. Run the narrowest useful validation first.
5. Stop when the requested outcome is met and checks are green.

## Verification Order

- Run the narrowest useful command first.
- Use `npm run typecheck` for TypeScript or contract changes.
- Use `npm run lint` for shared UI or hook changes.
- Use `npm run test` for behavior changes.
- Use `npm run doctor` only for Expo or native config changes.
- Use `npm run backend:cf:test` for Cloudflare backend changes.

## Token-Saving Rules

- Prefer path references over pasted code.
- Summarize diffs; do not restate unchanged code.
- Do not explain React, Expo, or TypeScript basics unless asked.
- Do not review unrelated files.
- Do not add speculative refactors, extra features, or extra edge cases after the task is already solved.
- Ask questions only if a missing product decision blocks a safe implementation.

## Review Mode

- Report the highest-signal issues first.
- Prefer the top 1-3 findings unless the user asks for a deep audit.
- Focus on bugs, regressions, contract mismatches, layout breakage, missing tests, and cost/rate-limit risks.
